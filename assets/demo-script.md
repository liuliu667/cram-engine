用极快节奏展示核心操作流程，节奏感 > 完整性。每个画面 3-5 秒。

| 时长 | 画面 | 文字/旁白 |

## B站/长视频版（3-5 分钟，深度展示）

### 0:00-0:30 — 开场 + 问题

画面：说话人（可出镜可不出镜）+ 屏幕录制

"上学期期末我用 AI 速成组织行为学。每回都要手打几百字提示词。烦。
我就想，能不能让 AI 真像个懂教育学的老师？
于是花了一个下午，写了 Claude Code Skill——期末速成引擎。

它不是那种把 PPT 转 PDF 笔记的工具。它是 AI 坐下来，一步一步教你。"

### 0:30-1:00 — 配置（展示交互式问答）

画面：输入 `/cram 社会学概论 start`，展示 7 步交互式问答过程

"你不需要手写任何配置文件。输入命令后引擎会问你 7 个问题——
什么课、用什么教材、有没有笔记、什么题型、哪些知识点。
两分钟答完，配置自动生成。

must_know 考点会在所有题型里全覆盖，key_points 抽样覆盖省时间。
知识点在 PDF 或 Word 里？直接复制粘贴过来就行。"

### 1:00-2:00 — 阶段 1+2 演示

画面：阶段 1 输出知识树 → 用户确认 → 阶段 2 实际讲一个知识点

"启动之后 AI 自动跑四个阶段。
阶段 1 把重点拆成知识树，每个点标注记忆钩子。
总数和分级分布展示给你，等确认再开始。

阶段 2 一个一个知识点讲——注意看，它不从定义开始。
它先丢生活场景，比如'你在小组作业里拼命干，因为你觉得努力能拿高分'。
然后讲核心逻辑，讲完反问你'你自己试试总结一下'。
每 3 个点暂停一次，你可以加速、减速、或者直接跳过进检题。
这才是教学。"

### 2:00-3:00 — 阶段 3+4 演示

画面：展示阶段 3 出题（多种题型）、用户做题、批改，阶段 4 诊断重讲

"阶段 3 按真实题型出题。选择题里藏陷阱项——干扰选项来自同一门课的其他知识点。
案例分析设问引导你：识别理论 → 解释机制 → 给建议。
批改是踩分点逐项对照，不跟你打马虎眼。
不管是选择、简答、论述还是名词解释，引擎自动适配。

阶段 4 只对做错的点补漏。先诊断：你是概念混淆了还是理解偏了？
然后换一种讲法重教。混淆给对比表，偏差给极端反例。
要是还错，标顽固点，建议明天再攻。
科学说的：间隔比死磕有效。"

### 3:00-3:30 — 总结 + 开源

画面：GitHub 仓库页面

"整个引擎：一个下午，7 个交互问题 + 四阶段 prompt + 一个 SKILL.md。
不需要几千万 token。
开源 MIT，链接在简介和评论区。

文史哲、法学、教育、新闻、社会学、经管……只要是定性知识多的课，
五分钟上手。期末不慌。"

## 录制要点

1. **屏幕录制**：OBS 或系统自带录屏，1920×1080，终端字体调大
2. **Claude Code 界面**：录制前把窗口背景调成深色，看着更专业
3. **提前跑一遍**：先试跑完整流程，确保网络和 API 正常，录的时候不卡
4. **剪掉等待时间**：AI 生成那几秒后期剪掉，只留输出结果
5. **字幕**：关键术语（认知负荷理论、精细加工、生成效应）打字幕，方便截图
6. **结尾**：GitHub 链接画面停至少 3 秒，方便截图
